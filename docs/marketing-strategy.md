# Snapmeal マーケティング戦略

> 作成日: 2026-05-22
> 対象: Snapmeal（snap-meal.com）ベータ版リリース後の獲得・拡大戦略
> 前提: ベータ版は無料・コンバージョン目標は「継続利用」と「フィードバック取得」

---

## 1. 市場環境と前提

### 1.1 市場の追い風

- **AI献立アプリ市場のCAGR: 約28.1%**（2024〜2030）。スマートフォン普及率の上昇とAI技術の浸透により急成長中
- **大手の空白**: Yummly が 2024年12月に終了（Whirlpool による買収後）。Cookpad は減退傾向、DELISH KITCHEN / Kurashiru は動画路線で躍進中
- **競合の特徴**: TastR AI、Grocery AI、Suphra、FoodiePrep など「冷蔵庫写真認識」「期限管理」「TikTokレシピ取り込み」を組み合わせたAIアプリが2025〜2026年にかけて急増

### 1.2 Snapmealのポジショニング

| 既存サービスの軸 | Snapmealの差別化 |
|---|---|
| クックパッド・DELISH = 大量レシピ閲覧 | **「献立を決める」体験そのもの** |
| Whisk / Yummly = 海外発・調味料不一致 | **日本の調味料事情・代替案の的確さ** |
| Mealime = 週間メニュー計画 | **「今夜どうする」即決即断** |
| 競合AI = 単機能 | **ホットクック × 多調理器具 × 疲れた日モード** の独自組み合わせ |

## 2. ターゲット定義

### 2.1 一次ターゲット（最初の100ユーザー）

**ペルソナA：ホットクック愛用層（30〜45歳）**
- ホットクックを所有しているが、献立決めに毎日10分以上悩んでいる
- レシピ本のレパートリーに飽きている
- シャープ公式「ホットクック部」コミュニティ会員、note・ブログ・Instagramで情報収集
- **獲得チャネル**: ホットクック部内での口コミ、note でのストーリー記事、ホットクック特化ブロガーへのアプローチ

**ペルソナB：共働き子育て世帯（30〜45歳・女性中心）**
- 平日18時に「今夜どうしよう」が日課
- 時短料理・冷蔵庫活用のキーワードでSNSをフォロー
- 1日5〜10分のスキマ時間でしかアプリを触らない
- **獲得チャネル**: Instagram リール、TikTok（共働き料理クリエイター）、X での共感投稿

### 2.2 二次ターゲット（フェーズ2以降）

- **一人暮らし層**：仕事帰りで疲れている層。「疲れた日モード」のフィット度が高い
- **高齢世帯・調理弱者**：シンプルなUI・大きい文字 → 後日UI調整必要
- **海外（後述）**：英語圏・韓国・台湾

### Phase 0：プレローンチ（〜2週間）

## 4. チャネル別戦術

### 4.1 コンテンツマーケティング（推奨度: ★★★★★）

レシピ分野の SEO は飽和しているが、**「献立決め」「冷蔵庫活用」「ホットクック × AI」**は競合が少ない。

**ターゲットキーワード例**
- 「ホットクック 献立 マンネリ」「ホットクック みりん 代用」
- 「冷蔵庫 写真 献立」「AI 料理 提案」
- 「疲れた日 料理 15分」
- 「鶏もも肉 キャベツ 献立」（食材組み合わせの長尾）

**コンテンツ形式**
- note: ストーリー＆開発裏話（共感）
- 自社ブログ（snap-meal.com/blog 新設推奨）: SEO狙いの実用記事
- YouTube/TikTok: 「撮影 → 提案 → 完成」30秒ループ

#### 4.1.1 SEO記事の量産設計（100記事）

結論として、SnapmealのコンテンツSEOは **100記事まで拡張可能**。ただし、最初から「レシピ記事」を大量に作るのではなく、以下の4カテゴリに分けて、検索意図ごとに記事テンプレートを固定する。

| カテゴリ | 記事数目安 | 狙う検索意図 | Snapmealへの導線 |
|---|---:|---|---|
| ホットクック × 食材 | 40記事 | 特定食材をホットクックでどう使うか知りたい | 「冷蔵庫の食材からホットクック向け献立を提案」 |
| 冷蔵庫食材 × 献立 | 30記事 | いま家にある食材で何を作れるか知りたい | 「食材入力ではなく写真でまとめて提案」 |
| 状況・悩み系 | 20記事 | 疲れた日、買い物に行けない日、平日夜の解決策が欲しい | 「疲れた日モード」「30秒で献立決定」 |
| 代用・調味料系 | 10記事 | 調味料や食材が足りないときの代替案が欲しい | 「不足食材・代替案も含めてAIが提案」 |

**初期は30記事を先行公開し、Search Consoleで表示回数が出た型へ追加投資する。** 100記事を一括公開するより、30記事 → 60記事 → 100記事の順で伸ばす方が、検索意図のズレを修正しやすい。

#### 4.1.3 100記事までの拡張案

初期30記事で反応を見た後、以下のように拡張する。

| 拡張フェーズ | 追加記事数 | 内容 |
|---|---:|---|
| 第2弾 | +30記事 | ホットクック × 食材を中心に、鶏むね肉、なす、もやし、豆腐、ブロッコリー、トマト、ツナ缶などへ展開 |
| 第3弾 | +25記事 | 食材2〜3個の組み合わせ記事を増やす。「卵 キャベツ 豚肉 献立」など |
| 第4弾 | +15記事 | 悩み系・代用系を増やす。「薄味」「子ども向け」「作り置き」「洗い物少ない」など |

#### 4.1.4 記事テンプレート

SEO記事は毎回構成を固定し、制作コストを下げる。

**ホットクック × 食材 記事**
1. 結論: その食材で作りやすい料理を3〜5個提示
2. ホットクックで向いている調理パターン
3. 失敗しやすい点（水分量、加熱時間、味付け）
4. 代用できる食材・調味料
5. 冷蔵庫に他の食材がある場合の献立例
6. CTA: Snapmealで冷蔵庫写真からホットクック向け献立を提案

**冷蔵庫食材 × 献立 記事**
1. 結論: その食材組み合わせで作れる献立を3つ提示
2. 主菜・副菜・汁物の組み合わせ例
3. 足りない場合に買い足すとよい食材
4. 調理器具別の作り分け（ホットクック / フライパン / レンジ）
5. CTA: 食材を入力せず、冷蔵庫を撮るだけで提案

**状況・悩み系 記事**
1. 共感: 夕飯決めがしんどい状況を言語化
2. 解決策: 判断を減らすルールを提示
3. 具体例: 15分・30分・ホットクック放置の3パターン
4. CTA: Snapmealの「疲れた日モード」へ誘導

#### 4.1.5 水曜制作の現実的な本数

水曜をコンテンツ制作日にする場合、1日で書ける本数の目安は以下。

| 品質ライン | 1日あたり本数 | 用途 |
|---|---:|---|
| 下書き量産 | 8〜12本 | AIで初稿を作り、後日編集する前提 |
| 公開可能な標準記事 | 3〜5本 | 検索意図確認、CTA、内部リンクまで整える |
| かなり丁寧な記事 | 1〜2本 | 競合調査、画像、実体験、独自性を入れる |

推奨は **毎週水曜に標準記事3本公開 + 下書き5本作成**。このペースなら、約10週間で30本公開、約6〜8ヶ月で100記事規模まで到達できる。

#### 4.1.6 内部リンク設計

記事単体で終わらせず、以下の導線を固定する。

- 全記事から `/` または `/beta` のベータ参加CTAへリンク
- ホットクック記事同士を「食材別ホットクックレシピ」カテゴリで相互リンク
- 冷蔵庫食材記事から「冷蔵庫にあるもので献立」まとめ記事へリンク
- 代用記事から「不足食材があっても献立を作る」系記事へリンク

#### 4.1.7 成果判定

公開後4週間は順位よりも **Search Consoleの表示回数** を見る。表示回数が出ているのにクリック率が低い記事はタイトル改善、表示回数が出ない記事は検索意図またはキーワード選定を見直す。

| 指標 | 初期目安 |
|---|---:|
| 30記事公開後の月間表示回数 | 5,000〜20,000 |
| CTR | 2〜5% |
| LP遷移率 | 3〜8% |
| ベータ参加CVR | 3〜10% |

#### 4.1.8 競合ペインポイント起点の記事戦略（2026年5月改訂・最優先施策）

> **背景**: [competitive-pain-analysis.md](./competitive-pain-analysis.md)（2026年5月）の分析により、「ログインユーザー2人」の主因は**トラフィック不足**であることが確認された。上記の「ホットクック×食材」「冷蔵庫食材×献立」記事は中長期のSEO基盤として有効だが、**転換意図が高い流入を最速で得るには競合の不満層を直接取りに行く記事が優先**。

**3層の記事優先順位**

| 優先度 | ターゲット層 | 理由 |
|--------|------------|------|
| **A（最優先）** | pecco・me:newに不満を持つユーザー | 課題認識済み・解決策を探している・転換意図が最高 |
| **B** | ChatGPTで献立を試みているユーザー | 自分で方法を探索中。「専用アプリの方が楽」と気づかせられる |
| **C（並行）** | 献立決めに疲弊している層全般 | 母数は最大だが転換まで時間がかかる。§4.1.2の記事群がここに相当 |

**優先度A：競合不満層を取り込む記事（今すぐ書く3本）**

1. 「peccoのデメリット・口コミまとめ｜ない食材のレシピが出る問題を解決したい人へ」
   - pecco最大の不満（存在しない食材が提案される）→ Snapmealは不足食材を明示して解決することを訴求
   - ターゲットキーワード：「pecco デメリット」「pecco 口コミ」「pecco 使えない」

2. 「me:newで同じ献立が繰り返される問題｜履歴学習アプリへの乗り換えガイド」
   - me:new最大の不満（週内で同じ食材・メニューが繰り返される）→ Snapmealの履歴学習を訴求
   - ターゲットキーワード：「me:new 同じ献立」「me:new デメリット」「献立アプリ マンネリ」

3. 「冷蔵庫を写真で撮るだけで献立が決まるアプリ比較【2026年版】」
   - Snapmeal・pecco・Frish・FridgeoAIを中立的に比較。ジャンルの認知獲得と差別化を同時に
   - ターゲットキーワード：「冷蔵庫 写真 献立 アプリ」「冷蔵庫 AI 献立」

**優先度B：ChatGPT比較記事（週1本ペースで書く2本）**

4. 「ChatGPTで献立を決める方法と限界｜毎回プロンプトを書く手間をなくす方法」
   - ChatGPT利用者の弱点：毎回入力が面倒、学習しない → Snapmealの1タップ起動・プロファイルを訴求
   - ターゲットキーワード：「ChatGPT 献立」「ChatGPT 料理 提案」「AI 献立 アプリ」

5. 「冷蔵庫の写真をChatGPTに送ると献立を提案してくれる？実際に試した結果」
   - 「試してみた」形式で中立的に検証し、「でも専用アプリの方がUXが良い」という結論へ着地
   - ターゲットキーワード：「ChatGPT 冷蔵庫 写真」「ChatGPT 献立 作り方」

**記事の共通構成（転換を最大化する型）**

```
1. 読者の悩みに共感（検索キーワードの背景にある感情を言語化）
2. 問題の深掘り（なぜそうなるか・競合の限界）
3. 解決策の選択肢を中立的に列挙（押し売りに見せない）
4. Snapmealの紹介（「こういう人に向いている」で限定的に見せる）
5. CTAは「無料で試す」1個のみ
```

**書いてはいけない記事（リソースを無駄にしない）**

- 「夕飯 献立 100選」「時短レシピ まとめ」→ Cookpad・DELISH KITCHENに永遠に勝てない
- 「献立 アプリ おすすめ」→ ランキング型まとめ記事に埋もれる
- 英語記事（現時点）→ 英語圏の競合密度は日本より高く、SEOで勝てるまで時間がかかりすぎる
- 「ホットクック レシピ ○○」単体記事のさらなる量産 → 基盤は整っており、投資対効果が逓減している

**最初の1本の選定根拠**

> **「peccoのデメリット・口コミ｜ない食材のレシピが提案される問題とSnapmealの違い」**
>
> peccoは50万DL・App Storeに「持っていない食材のレシピが出される」という不満レビューが多数。このユーザーは課題認識済みで代替を探している。転換意図が最も高い流入を最速で獲得できる。

### 4.3 コミュニティ（推奨度: ★★★★★）

**シャープ公式「ホットクック部」**

### 4.4 インフルエンサー（推奨度: ★★★☆☆）

**コスト効率の良いマイクロ層**
- フォロワー1万〜10万の共働き料理 / 時短系
- 例：ぐっち夫婦（共働き）、ゆぅゅぅ（時短節約）など類似ペルソナを5〜10名選定
- 報酬：ベータ期間中は Pro 無料 + 物販ギフト程度

**避けるべき**
- 大手インフルエンサーへの一括投資（CPA合わず・継続性なし）
- 単発タイアップ（ベータ期は継続的な関係構築重視）

### 4.5 Product Hunt（推奨度: ★★★★☆）

- 1度きりだが、英語圏ユーザーへの認知獲得に効果絶大

### 5.1 市場選定の優先順位

| 優先度 | 市場 | 理由 |
|---|---|---|
| 1 | 🇰🇷 **韓国語** | 共働き世帯比率が高い、家電・AI親和性が高い、料理スタイル類似 |
| 2 | 🇹🇼 **繁体字中国語（台湾）** | 共働き・少子化・親日市場、人口比に対しARPU高い |
| 3 | 🇺🇸 **英語圏** | TAM最大だが競合激戦区。Product Hunt経由の自然流入活用 |
| 4 | 🇪🇸 🇲🇽 **スペイン語** | 中南米含めTAM大、ローカライズコスパ良 |
| 5 | 🇨🇳 **簡体字中国語** | 規制リスク高、後回し |

### 5.2 ローカライズの注意点

**翻訳だけでは足りない（カルチャライゼーション）**
- 韓国：辛味・発酵・即席料理スタイルへの対応（キムチ・ラーメン・ポッサム等）
- 台湾：朝食文化・夜市料理・電鍋（タイワン版ホットクック）対応
- 英語圏：オーブン文化、計量単位（cup/oz）、調味料体系の差

**研究結果**
- ローカライズしたアプリは英語のみと比較して **DL数 +128%、収益 +26%**（業界統計）
- 日本国外ユーザーがアプリを判断する最初の数秒は「自分の言語か」が最重要

### 5.3 ローカライズの実装方針

| 項目 | 対応 |
|---|---|
| UI翻訳 | next-intl の既存基盤を拡張。AI翻訳（DeepL/GPT）+ ネイティブレビュー |
| AIプロンプト | 各言語で食材名・調味料・料理スタイルを別々に最適化 |
| LP | snap-meal.com/ko/ snap-meal.com/zh-tw/ など順次追加 |
| OGP / メタ | 各言語別に専用画像生成 |
| GA4 | 言語別レポート設計、`country` ディメンションで KPI 追跡 |

- **オーガニック率**: 広告依存度の指標。理想は 70% 以上

## 9. やらないこと（重要）

リソース集中のため、当面は以下を**意図的に避ける**：

- **Google/Facebook 広告**: CAC が読めない段階で予算投入しない
- **大手インフルエンサー一括契約**: ROI不透明、ベータ期には不適
- **App Store / Google Play 正式リリース**: PWA で当面十分。テスター12人要件回避
- **過度な機能追加**: ベータ期はコア体験の検証に集中。要望は記録するが実装は精査
- **複数言語の同時拡大**: 韓国語 → 英語 → 中国語の順で1つずつ仕上げる
